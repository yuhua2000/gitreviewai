# 第二阶段需求

## 1. 添加前端页面

## 实现登录
1. 这块先简单实现，用一个写死的密码即可（在配置中），不用用户名
2. 要登录才能进入其他页面

## 合并请求管理
1. 实现合并请求列表
2. 实现合并详情
  - 可以查看合并情况详情，变动文件详情和评论详情。
  - ai生成的评论和报告 需要人工管理才可以提交。
  - 设置自动提交设置，设置允许不通过人工即可自动提交ai生成的评论和报告
  - 评论和报告的显示 使用 markdown 格式渲染，
  - 已经提交的不能重复提交，要展示为已提交
  - 相关评论可以展示到相关变动附近（可选实现 ui 参考 gitlab 合并请求页面）

## 技术实现

1. 可以在当前 审计完合并请求，讲数据和合并请求一并存入数据库（暂且用sqlite），或者先存入数据库 在进行审计，将审计结果存入数据库
2. 如果设置了自动提交，除了将当前 评论和报告存入数据库 还需要自动提交
3. 前端用vue实现 ，页面要精简漂亮，合理使用一些前端ui库和框架
4. 登录虽然只有密码，需要走完整登录流程，后面的操作要鉴权。
5. 前端不单独用 nodejs 启动，也用当前程序提供前端页面。 
6. 设计好完整的数据库表，合并请求表，评论，报告表
7. 要记录下评论和报告的id，方便后续新增修改和删除功能
