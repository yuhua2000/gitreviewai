# GitLab Merge Request Review 机器人需求文档

## 1️⃣ 项目概述

**目标**：
开发一个自动化机器人，用于 GitLab Merge Request (MR) 的代码审计和 Review，自动生成行级别评论，并提供完整的 Markdown 总结，帮助团队提高代码质量和安全性。

**核心功能**：

* 自动拉取 MR diff 或修改文件
* AI 审核代码修改，生成建议
* 提交行级别评论到 MR 页面
* 提交综合 Markdown 总结
* 可配置审计规则、AI 模型、提交策略

---


## 2️⃣ 功能需求

### 2.1 MR 事件触发

**触发条件**

* MR 创建
* MR 更新（push commit）

**实现方式**

* GitLab Webhook 调用机器人服务
* 支持异步处理，避免阻塞 GitLab

---

### 2.2 获取 MR 内容

**功能描述**

* 拉取 MR 变更的文件列表和 diff
* 获取每个文件的新增/修改行号

**接口**

* GitLab API: `GET /projects/:id/merge_requests/:mr_iid/changes`
* 支持 checkout 完整分支，以便 AI 分析上下文

---

### 2.3 AI 审核代码

**功能描述**

* 对 diff 或完整文件进行分析
* 支持以下类型审计：

  * 代码风格
  * 潜在 bug
  * 安全漏洞
  * 性能优化
  * 文档缺失
* 输出：

  * 行级建议（file, line, message, severity）
  * MR 总结（Markdown 格式）

**实现方式**

* 可选本地 AI 模型或云端 API
* 支持可配置规则模板和提示词

---

### 2.4 提交行级评论

**功能描述**

* 对每个审计问题提交评论到 MR 对应行
* 支持 `new_line` 或 `old_line` 精确定位

**接口**

* GitLab API: `POST /projects/:id/merge_requests/:mr_iid/notes`
* `position` 字段用于行级别评论

---

### 2.5 提交 MR 总结

**功能描述**

* 将 AI 生成的 Markdown 总结作为 MR 评论提交
* 可包含：

  * 审核概览
  * 风险列表
  * 修改建议

**接口**

* 与行级评论接口相同，作为普通 comment 提交

---

### 2.6 可选：自动审批或标记

**功能描述**

* 可自动标记 MR 状态，例如 `approved` 或 `needs work`
* 建议保留人工最终确认

---


## 3️⃣ 技术方案

### 3.1 架构设计

* **触发器**：GitLab Webhook → Robot Service
* **审核流程**：

  1. 获取 MR diff
  2. AI 审核（生成行级问题 + MR 总结）
  3. 提交行级评论
  4. 提交 MR 总结 Markdown
* **数据存储**：

  * 临时缓存 diff 和 AI 输出
  * 可选持久化 MR 审核记录

### 3.2 接口与协议

* GitLab API：

  * 获取 MR diff
  * 提交行级评论
  * 提交 MR 总结
* AI 接口：

  * 输入：文件 diff / 文件内容
  * 输出：行级问题 + Markdown 总结

### 3.3 配置项

* GitLab：

  * Token
  * MR 项目 ID
* AI：

  * 模型路径或 API Key
  * 审核规则模板
* 审核策略：

  * 行级 comment 最多数量
  * 总结提交策略（每次 commit 或 MR 合并前）
  * 忽略文件/目录列表

## 7️⃣ 技术选型

**实现语言**

* Go 1.26.2
* 并发处理 MR 审核和评论提交，提高性能

**模块名称**

* `github.com/yuhua2000/gitreviewai`
* 统一管理项目依赖和工具接口

**AI 对话与模型集成**

* 使用 `github.com/openai/openai-go` 调用 OpenAI API
* 支持多轮对话（multi-turn conversation）
* 提供工具接口（Tools）给大模型调用

工具接口调用约束

- 不在提示词中模拟工具调用
- 使用模型自带的工具能力（tooling API / function-calling）
- 向模型注册工具列表（Tool Definitions）

**提供给 AI 的工具接口（Tools）**

| 工具名称                                  | 功能说明                                 | AI 调用方式       |
| ------------------------------------- | ------------------------------------ | ------------- |
| `AddLineComment(file, line, message)` | 生成行级 comment 建议（AI 调用，不直接提交 MR）      | AI 调用 tool 参数 |
| `AddReviewComment(message)`           | 生成 MR review 建议（AI 调用，不直接提交 MR）      | AI 调用 tool 参数 |
| `GenerateMDReport()`                  | 整理本次 MR 的 Markdown 审核报告（最终由机器人提交 MR） | AI 调用 tool 参数 |
| `ReadFile(path)`                      | 读取指定文件内容                             | AI 调用 tool 参数 |
| `FindInFile(path, pattern)`           | 在文件中查找内容                             | AI 调用 tool 参数 |
| `GetURL(url)`                         | 获取指定 URL 内容（用于参考或文档）                 | AI 调用 tool 参数 |

**扩展性与可维护性**

* 工具接口可扩展（文件分析、复杂搜索、依赖分析等）
* 支持替换 AI 模型或修改对话规则
* Go 语言强类型保证接口安全和可维护性

## 8️⃣ 审核流程设计

### 8.1 流程概览

1. **Webhook 触发 MR 审核**

   * MR 创建或更新
   * 机器人拉取 MR 的 commit 信息、文件树及 diff

2. **初始化 AI 审核对话**

   * 将 commit 信息和文件树作为初始上下文输入 AI
   * AI 了解本次 MR 的整体结构

3. **多轮对话 + 工具接口**

   * AI 调用工具接口获取详细信息：

     * `ReadFile(path)` → 读取文件内容
     * `FindInFile(path, pattern)` → 查找关键内容
     * `GetURL(url)` → 获取外部参考文档
     * `AddLineComment(file, line, message)` → 生成行级评论建议
     * `AddReviewComment(message)` → 生成 MR review 建议
   * AI 可以逐步迭代分析，生成更准确的建议

4. **机器人处理 AI 输出**

   * AI 工具调用的结果由机器人统一收集：

     * 行级 comment 建议
     * MR review 建议

5. **统一生成 Markdown 报告**

   * 使用 `GenerateMDReport()` 整理：

     * 审核概览
     * 风险列表（文件+行+问题）
     * 修改建议
   * 报告作为单独 Markdown 文件或 MR 评论提交

6. **最终提交**

   * 机器人统一执行：

     * 提交行级 comment
     * 提交 MR review
     * 提交 Markdown 报告到 MR 页面
   * AI 只提供建议，不直接操作 GitLab
